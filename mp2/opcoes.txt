Grupo 09 - Opções:
Neste trabalho para fazermos o alisamento dos unigramas e bigramas decidimos usar o alisamento de LaPlace, devido à sua simplicidade (não foi dito para usar um específico). Esta técnica de alisamento tem a vantagem de ser simples, mas a desvantagem de mover uma grande porção da massa probabilística dos eventos vistos para os não vistos (não apresenta os melhores resultados).
Durante o trabalho assumimos que temos exactamente dois lemas para a palavra que procuramos (não generaliza para palavras com outras quantidades de lemas possíveis).
Para cada lema criamos um ficheiro com os unigramas e a respectiva contagem, outro com os bigramas e respectiva contagem, e outros dois que são o equivalente aos primeiros mas com o alisamento aplicado às contagens. Criado usando python compute_n_grams.py. Os ficheiros escritos têm o formato lema(Alisamento)?\.(unigramas|bigramas), por exemplo serAlisamento.unigramas é o ficheiro com a contagems de unigramas para o lema ir, com alisamento. Apenas os ficheiros com alisamento foram incluidos na entrega, porém o script run.sh cria todos os ficheiros.
No enunciado são pedidos os ficheiros com a contagems de unigramas e bigramas, ambas com alisamento. Estes ficheiros estão na entrega com os nomes pedidos (diferentes daqueles que o programa cria).
No caso dos unigramas, decidimos colocar o simbolo UNK, que simboliza o desconhecido no corpus (aquilo que foi visto zero vezes). Para os bigramas o desconhecido é calculado no programa porque no caso de termos UNK foram por exemplo a contagem com alisamento seria 0 na mesma (multiplicado por contagem de UNK tudo dá zero).
No corpus que usámos para treinar o modelo não contámos com as frases que tinham uma classificação diferente dos lemas disponíveis, ou seja, não contamos com frases cuja classificação fosse n-é-verbo ou frases com dúvidas na anotação.
Para ver qual a classificação mais provável para uma frase (ou seja, para a palavra nessa frase) consideramos apenas os bigramas que contêm essa palavra e não calculamos a probabilidade da frase inteira. Por este motivo no ficheiro de bigramas também só contamos com aqueles que tenham a palavra que procuramos classificar.
As frases de teste não foram retiradas do corpus porque os resultados obtidos seriam muito optimistas (o modelo foi treinado com essas frases).
O script run.sh cria todos os ficheiros necessários à utilização do programa lema.py. Os ficheiros com a contagem de unigramas sem alisamento não foram incluidos na entrega mas são gerados durante a execução do script.
